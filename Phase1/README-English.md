## Overview

Aurora-Siger simulates the complete pre-launch verification cycle of an aircraft. At each execution, random values are generated for onboard telemetry and energy autonomy variables. Each variable is compared against its nominal operational range and assigned an individual status. The final launch decision is only approved if **all** parameters meet the defined thresholds.

---

## Features

- Random generation of telemetry data (temperature, pressure, structural integrity, critical modules)
- Energy autonomy calculation based on capacity, current load, consumption, and losses
- Individual parameter evaluation with `[ OK ]` or failure status
- Automated launch decision: `READY FOR TAKEOFF` or `TAKEOFF ABORTED`
- Interactive terminal output with step-by-step pauses

## Code Structure

```
FIAP_Aurora_Siger_Project.ipynb
├── generate_telemetry()     # Generates and evaluates telemetry parameters
├── calculate_autonomy()     # Generates and evaluates energy parameters
└── main program             # Calls functions, decides, and displays results
```

### `generate_telemetry()`

Randomly generates onboard variables and evaluates each one against operational thresholds.

Returns two dictionaries: `telemetry_data` and `telemetry_status`.

---

### `calculate_autonomy()`

Calculates aircraft energy autonomy based on generated and derived values.

Returns two dictionaries: `autonomy_data` and `autonomy_status`.

## Parameters and Thresholds

| Parameter              | Approval Condition        |
|------------------------|---------------------------|
| Internal temperature   | 21 °C <= value <= 30 °C   |
| External temperature   | -5 °C <= value <= 30 °C   |
| Structural integrity   | value == 1                |
| Tank pressure          | 95 <= value <= 145 bar    |
| Critical modules       | value == 1                |
| Battery charge         | >= 80%                    |
| Usable energy          | >= 10 kWh                 |
| Energy loss            | <= 5%                     |
| Autonomy               | >= 10 months              |

---

## Decision Logic

After evaluating all parameters, the `telemetry_status` and `autonomy_status` dictionaries are combined. If **all** values are `[ OK ]`, launch is approved. Any failure results in takeoff cancellation.

```python
decision = 'READY FOR TAKEOFF' if all(
    s == '[ OK ]' for s in {**telemetry_status, **autonomy_status}.values()
) else 'TAKEOFF ABORTED'
```
